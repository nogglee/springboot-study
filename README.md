### 🎯 Learning Goal
- 스프링 부트 기반 웹 애플리케이션의 전체 흐름(요청 → 처리 → 응답)을 코드 레벨에서 설명할 수 있다.
- MVC 구조에서 Controller · Service · Repository의 책임 분리 이유를 명확히 이해한다.
- DI / IoC를 적용해 구현체 교체가 가능한 구조를 직접 설계·구현한다.
- 테스트 코드를 통해 비즈니스 로직을 검증하는 습관을 체화한다.
- JDBC → JdbcTemplate → JPA → Spring Data JPA로 이어지는 DB 접근 방식의 진화 이유를 비교 설명할 수 있다.
- 트랜잭션과 AOP를 통해 핵심 관심사와 공통 관심사를 분리할 수 있다.
